# Daily-note seeding lives at the open boundary

Status: accepted (2026-07-08)

Opening a daily note "to write" (the `/today` route, the header **Today** button, the Cmd+K **"Go to Today"** action) seeds ONE empty entry line and lands the caret on it, so a journal-style note opens ready to append instead of on its date title. Every OTHER path to a day (**Send to Today**, **Mirror to Today**, the MCP daily tools, a date-chip click) leaves the day's structure untouched. The seed is an **opt-in at the open boundary**, never an invariant of `ensureDay`. Grew out of PR [#100](https://github.com/cameronapak/dotflowy/pull/100)'s `/today` route.

- **The principle: a daily note is an entry point, not an invariant.** There is no rule that a day always holds a blank bullet. An empty day stays empty (a bare title-only node) until a _write-intent_ surface opens it. This is why `ensureDay`/`getOrCreateDay` are deliberately **seed-free**: seeding at the get-or-create would fan an empty bullet into every caller that only wants the id (the whole blast radius below), and would re-materialize a bullet on a day the user intentionally emptied.
- **Opt-in shape**: `getOrCreateDay(key, index, { seedEntryLine?: boolean })` → `ensureDay(..., seedEntryLine)`. Only the three write-intent callers pass `true`.
  - **Fresh day** → the seed `appendChild(dayId, null, '')` rides the **same** `runStructural` batch as the day node (correct-by-construction; no sibling-chain fan).
  - **Emptied day reopened via a write-intent surface** → re-seeded in its own isolated batch (accepted). The childless check reads the **live** `nodesCollection.toArray` (`hasChildInLiveCollection`), **not** the passed `TreeIndex`, which can predate a just-fired seed (the tree-store index is a module singleton synced async; the daily plugin already treats `ctx.tree` as stale post-get-or-create — same discipline as `runMany`/mirror).
  - No `capture()` — the seed stays out of undo history, consistent with day creation itself.
- **Focus-last, unified across the three write-intent surfaces.** A plain route navigation has no zoom pivot, so `OutlineEditor`'s zoom-focus effect never fires and the caret would land nowhere. The `?focus=last` search param (validated through `validateOutlineSearch` in `src/data/tags.ts` — the destination route validates search, so an unlisted key is silently dropped) drives the `!pivotId && focusLast` branch of the editor's mount-only focus effect, landing the caret on the day's **last visible child** (the seeded line). The Today button drops `ctx.nav.zoom` for this path and navigates the route (`useNavigate` → `/$nodeId?focus=last`); the Cmd+K action bumps `SearchActionContext.goTo(id, { focus?: 'last' })` (Seam J). A date chip keeps `goToDate`'s zoom-and-no-seed (a reference, not a write).
- **Why the seed matters at all**: a zoomed day's title is a live contentEditable whose text IS the date; typing there corrupts it (Enter on the title adds a first child). A guaranteed empty child gives the caret a safe home and makes "open today, start typing" work with zero footguns.

**Do not "simplify" the seed back into `ensureDay`.** The seed-free get-or-create is load-bearing precisely because its callers diverge: Send/Mirror-to-Today and the MCP path (`planEnsureDaily`/`planAddToDaily`, already seed-free) must leave a clean structure when they move or copy content in — a stray empty bullet above moved children is the bug the old create-path seed and reopen-backfill caused across that whole set.

Rejected: **seed inside `ensureDay` for all callers** (fans a bullet into every reference path; the blast radius above). Rejected: **an invariant that a day always has a blank bullet** (fights intentional emptiness; needs a heal on every read). Rejected: **hard reload / `window.location.replace` for `/today`** (tears down the Effect runtime + sync socket and aborts the in-flight structural write — `runStructural` resolves on the optimistic insert, not the durable `POST /api/nodes {ops}` — so the day would vanish; the route stays an SPA navigation).
