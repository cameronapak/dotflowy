# Inline emphasis as folding tokens

Inline emphasis -- `*italic*`, `**bold**`, `~~strikethrough~~`, `~underline~` (Bear-style) -- is parsed from `node.text` (no schema field) and rendered as a **folding token**, modelled on the rich-link fold ([ADR 0005](./0005-rich-links-source-offset-caret.md)): a run shows its raw markdown markers only when the caret is within or adjacent to it, and folds to a clean styled tag (`<em>`/`<strong>`/`<del>`/`<u>`) otherwise. v1 is flat: no nesting, no `***bold+italic***`, no `_underscore_` variants -- an unmatched or nested attempt renders as literal text.

**Emphasis is a link, not a new token shape.** The intended UX is Bear/Obsidian live-preview: the markers hide, and when the caret touches a run the markers reveal as **real, walk-through editable text** -- the caret steps through `*italics*` one character at a time and can land inside the fence (`*italics|*`). That is *exactly* the rich-link bracket reveal ([ADR 0005](./0005-rich-links-source-offset-caret.md)), minus the URL atom. So emphasis reuses the existing folding-token machinery verbatim: each kind is a `TokenSpec` with `folds: true`, and the render switches on `revealOffset ∈ [start, end]` (boundaries inclusive) just like the link. No new `TokenSpec` flag, no new caret math.

**Folded run = one atom (`data-src`), NOT an interior-editable span.** The rejected alternative kept the interior text always-editable and made only the boundary markers virtual (hidden `data-edge` attrs, revealed in place). It reads better on paper -- you could click into the middle of a bolded sentence without the markers showing -- but it would require building the single most delicate subsystem in the app, the source-offset caret math *with virtual edges* (what ADR 0005 calls "the landmine"), from scratch: `readSource`, `getCaretOffset`/`setCaretOffset`, `getSelectionRange`, copy/cut, the slash/tag menus, and Enter-split would each need a new virtual-edge-counting path, and every line carrying emphasis would route through unproven code. Instead the folded run is `<em data-src="*italic*" contenteditable="false">italic</em>` -- the **exact atom shape a folded link already uses**, keyed on `data-src` generically (ADR 0001 D6). `readSource` reconstructs `*italic*` and the caret helpers count the atom's length with **zero new machinery**. The cost is small and mostly erased by reveal-on-proximity: while folded, the interior is opaque (a mid-word click inside a run reveals first, then lands on a second click) -- but the moment the caret is in the run, the markers show *anyway*, which is the whole feature. Robustness (reuse the battle-tested atom path) beat the marginal editing nicety.

**Revealed markers are dimmed and OUTSIDE the styled span.** A revealed run is `<span class="emphasis-reveal">·*·<em>italics</em>·*·</span>` where the markers are `.md-punct` spans (the same dimmed class the link reveal uses for `[]()`). Markers sit outside the `<em>`/`<strong>`/etc. so they read as plain syntax scaffolding, not slanted/bold content, and -- the load-bearing part -- the revealed state is **pure real text with no `data-src`**, so the caret walks through it with no special-casing. The virtual-atom path is confined to the *folded* state only.

**Bear-style syntax, chosen deliberately.** All four are Bear's markup (`bear.app` docs: `**bold**`, `*italic*`, `~underline~`, `~~strikethrough~~`, created via ⌘B/⌘I/⌘U/⌘⇧U). `*`/`**`/`~~` are also CommonMark/GFM standard; **underline has no markdown standard at all** (CommonMark left it out; the web fallback is raw `<u>` HTML). Single-tilde `~underline~` is the one real trade-off: it collides with the everyday "approximately" tilde (`~$5` in prose can mis-tokenize) and is forever-coupled to `~~strikethrough~~`'s double tilde. Accepted because (a) the user chose Bear-parity, (b) creation is hotkey/`/command`-driven so the tilde is rarely hand-typed, and (c) it keeps the raw form terse. The `~`/`~~` and `**`/`*` overlaps are resolved by precedence: bold (30) and strikethrough (31) sit **before** italic (32) and underline (33) in the combined regex, so the double-char marker always wins on an overlapping span. This is the only place in the token system where one syntax is a prefix of another.

**Don't:**
- Reintroduce the "edged token" idea (virtual CSS `::before`/`::after` markers, per-edge reveal). It can't put the caret *between* the interior and a marker (the marker isn't a real caret stop), which is the exact behavior this feature is *for*. The revealed markers must be real text.
- Treat `el.textContent` as the source once a line holds emphasis -- a folded run shows only its interior, so `readSource` (which reads `data-src` for the atom) is the source, just as for folded links.
- Add nesting, `***triple***`, or `_underscore_` variants without rewriting the flat single-pass tokenizer -- a recursive parse is a real rework, not an extension.
- Add a `<u>`-from-HTML or rich-HTML-paste conversion path in v1; paste stays plain-text. Smart auto-pairing on type (typing `*` yielding `**` with the caret inside) is deferred too -- creation is Cmd+B/I/U + Cmd+Shift+X + the `/` commands, and typed markdown tokenizes on its own once a run closes.
- Generalize the `~`/`~~` and `**`/`*` disambiguation into a "shared leading char" abstraction -- two cases don't justify it, and the precedence numbers encode the rule inline.

**Inline code folds the same way.** `` `code` `` (the `code` plugin) was flipped from a non-folding token (backticks always visible) to this same folding model -- backticks hide, and reveal as dimmed `.md-punct` text INSIDE the styled `<code>` box on caret proximity (inside, not flanking, because code has a visible container -- emphasis keeps its markers outside its bare tag). It's the same `folds: true` + atom-when-folded shape, so it reuses the identical machinery (no `inline-code.ts` change beyond the shared `hasLink → hasFoldingToken` guard already made for emphasis). Covered by `e2e/inline-code.spec.ts`.

**Search + display flatten emphasis.** `flattenInline` (`src/data/inline-text.ts`) composes `stripEmphasis(stripLinks(text))` -- the one funnel the node switcher and mirror-places use so a search index or a switcher title never shows raw `*markers*`. `e2e/emphasis.spec.ts` covers the fold/reveal transition, the arrow-key walk into the fence across all four kinds, and hotkey/slash creation.
