# changelog/

The **archived changeset fragments** — the app's changelog data (ADR 0046).

`changeset version` _consumes_ the fragments in `.changeset/`, so `bun run release`
copies them here first, one directory per released version, and appends the version

- date to `manifest.json` (chronological, oldest first).

Two things read this directory, and neither reads the other's output:

- **`scripts/vite-plugin-changelog.ts`** parses + Effect-Schema-validates it at build
  time and compiles the result into the bundle. A malformed entry fails the build.
- **`CHANGELOG.md`** (repo root, generated by changesets) is the input to the GitHub
  Release. It is prose for humans and is never parsed back into app data.

**Never run `changeset version` directly.** It deletes the fragments before they are
archived, and `bun run build` will then fail because `changelog/<package.json version>/`
does not exist. That failure is the process.
